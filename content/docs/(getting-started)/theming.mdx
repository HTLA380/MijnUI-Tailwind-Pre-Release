## Creating your own theme

While Mijn-UI doesn't currently have a dedicated theme builder, you can still create your own theme by following these simple steps:

<Steps>

<Step>
  Visit the [Shadcn UI Theme
  Generator](https://zippystarter.com/tools/shadcn-ui-theme-generator) to design
  your theme. Once you're done, copy the generated CSS variables.
</Step>

<Step>
 Next, you'll need to modify some of the variables to align with Mijn-UI's structure. For example, Mijn-UI uses the `--surface` variable for both cards and popovers. Here’s how to replace them:

```css
-   --card: 343.4 50% 90%;
-   --card-foreground: 343.4 5% 10%;
-   --popover: 343.4 100% 95%;
-   --popover-foreground: 343.4 100% 0%;

+   --surface: 343.4 50% 90%; /* [!code highlight] */
+   --surface-foreground: 343.4 5% 10%; /* [!code highlight] */
```

Rename the `--destructive` variables to `--danger` to follow Mijn-UI’s naming convention:

```css
-   --destructive: 0 100% 30%;
-   --destructive-foreground: 343.4 5% 90%;
-   --radius: 0.5rem;

+   --danger: 0 100% 30%; /* [!code highlight] */
+   --danger-filled-foreground: 343.4 5% 90%; /* [!code highlight] */
```

Finally, add any additional colors for states like `success`, `warning`, `danger`, and `info`. Here’s an example:

```css
--info: 216 76% 52%; /* Blue for informational state */ /* [!code highlight] */
--info-foreground: 219 64% 43%; /* Darker blue for better contrast */ /* [!code highlight] */
--info-filled-foreground: 0 0% 100%; /* White for filled backgrounds */ /* [!code highlight] */

--danger: 0 72% 43%; /* Red for danger state */ /* [!code highlight] */
--danger-foreground: 0 82% 31%; /* Darker red */ /* [!code highlight] */
--danger-filled-foreground: 0 0% 100%; /* White for filled backgrounds */ /* [!code highlight] */

--warning: 45 92% 47%; /* Yellow for warnings */ /* [!code highlight] */
--warning-foreground: 28 73% 26%; /* Darker yellow */ /* [!code highlight] */
--warning-filled-foreground: 0 0% 0%; /* Black for filled backgrounds */ /* [!code highlight] */

--success: 140 76% 38%; /* Green for success */ /* [!code highlight] */
--success-foreground: 140 49% 20%; /* Darker green */ /* [!code highlight] */
--success-filled-foreground: 0 0% 100%; /* White for filled backgrounds */ /* [!code highlight] */
```

And that's it! You've successfully created and customized your own theme for Mijn-UI.

</Step>

</Steps>

## Color Conventions

`background` and `foreground` colors

- The `background` variable is used for the background color of the component.
- The `foreground` variable is used for the text color.

### Example

For the following CSS variables:

```css title="Css"
{
  --primary: 29 100% 52%; /* orange-500 */
  --primary-foreground: 0 0% 100%; /* white */
}
```

- The background color will be `hsl(var(--primary))`.
- The foreground (text) color will be `hsl(var(--primary-foreground))`.

#### Usage in the Component

```tsx title="example.tsx"
<div className="bg-primary text-primary-foreground">Hello</div>
```

## Adding New Colors

To add new colors, you need to add them to your CSS file and to your tailwind.config.js file.

<Tabs items={['global.css', 'tailwind.config.js']}>
  <Tab value="global.css">
  ```css
  :root {
    --card: 0 0% 98%; /* neutral-50 */
    --card-foreground: 0 0% 0%; /* black */
  }

.dark {
--card: 0 0% 7%; /_ #131313 _/
--card-foreground: 0 0% 96%; /_ neutral-100 _/
}

````
  </Tab>

  <Tab value="tailwind.config.js">
  ```js
  module.exports = {
    theme: {
      extend: {
        colors: {
          card: "hsl(var(--card))",
          "card-foreground": "hsl(var(--card-foreground))",
        },
      },
    },
  }
````

  </Tab>
</Tabs>

You can now use the new colors in your components.

```tsx
<div className="bg-card text-card-foreground">Hello</div>
```

### Other Color Formats

While we use HSL colors for theming as recommended by [ShadCN](https://ui.shadcn.com/docs/theming). However, you can also use other color formats if you prefer. Tailwind CSS supports various color formats including RGB, RGBA, and HSL.

For more information on using different color formats, refer to the [Tailwind CSS documentation](https://tailwindcss.com/docs/customizing-colors#using-css-variables).
